## Backend (FastAPI)

```powershell
cd D:\CODEX\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# Set env vars (see .env.example) then run
uvicorn app.main:app --reload
```

## Frontend (React + Vite)

```powershell
cd D:\CODEX\frontend
npm install
npm run dev
```

Open http://localhost:5173 and make sure `VITE_API_URL` points to the backend.

## GitHub Pages (Static Frontend)

1. Create a GitHub repo named `GETTICKET` under `NICKIE887`.
2. Push this project to the repo's `main` branch.
3. In GitHub repo settings, go to Pages and set the source to `GitHub Actions`.
4. Wait for the workflow to finish.

Your URL will be:
- https://nickie887.github.io/GETTICKET/

## Demo Mode

When the frontend cannot reach the backend API, the Events page falls back to demo events so the homepage is not empty. This is automatic and only affects the events list view.

## Database Migration (Safe)

The backend auto-applies safe SQLite migrations at startup for newly added columns such as:
- `events.image_url`
- `tickets.contact_phone`
- `tickets.contact_email`

No manual migration is required for local use.

## Image Uploads

Event images can be uploaded directly to the backend.

- Endpoint: `POST /uploads/event-image`
- Requires login
- Returns a public URL like `http://localhost:8000/uploads/<file>`

## Deploy the Backend (FastAPI)

You can host the backend on Render (free tier) in a few minutes.

1. Create a Render account and click **New Web Service**.
2. Connect the GitHub repo `NICKIE887/GETTICKET`.
3. Set the root directory to `backend`.
4. Build command:
   ```
   pip install -r requirements.txt
   ```
5. Start command:
   ```
   uvicorn app.main:app --host 0.0.0.0 --port 10000
   ```
6. Add environment variables from `backend/.env.example` (set real values for M-Pesa). Ensure:
   - `DATABASE_URL=sqlite:///./app.db`
   - `CORS_ORIGINS=https://nickie887.github.io`
7. Deploy and copy the public backend URL (e.g. `https://gettickets-api.onrender.com`).

## Wire VITE_API_URL

After you have the backend URL, update the frontend build variable.

1. Edit `frontend/.env.example` or set `VITE_API_URL` in your GitHub Actions workflow/environment for production.
2. For GitHub Pages, the easiest approach is to commit a new `.env` file in `frontend`:
   ```
   VITE_API_URL=https://your-backend.onrender.com
   ```
3. Rebuild and redeploy the frontend (push to `main`).

## Notes
- M-Pesa STK push requires a public callback URL and valid Daraja credentials.
- Ticket PDFs are generated by printing the ticket view from the browser.
